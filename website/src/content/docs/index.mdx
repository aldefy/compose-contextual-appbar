## What Is It?

`compose-contextual-appbar` provides a top app bar that **switches between a default state and a contextual action mode** when the user selects items. You've seen this in Gmail, Google Photos, and most Google apps — select an email and the toolbar transforms to show delete, archive, and move actions.

This library brings that pattern to Compose Multiplatform with smooth animated transitions, back-press handling, and full Material 3 support.

## Quick Install

```kotlin
// build.gradle.kts
commonMain.dependencies {
    implementation("io.github.aldefy:contextual-appbar:1.0.0-alpha01")
}
```

## Quick Example

<Tabs>
  <TabItem label="Material (recommended)">
    ```kotlin
    var selectedIds by remember { mutableStateOf(emptySet<Int>()) }

    Scaffold(
        topBar = {
            MaterialContextualTopAppBar(
                selectedCount = selectedIds.size,
                onClearSelection = { selectedIds = emptySet() },
                defaultBar = {
                    TopAppBar(title = { Text("Inbox") })
                },
                contextualActions = {
                    IconButton(onClick = { /* delete */ }) {
                        Icon(Icons.Default.Delete, contentDescription = "Delete")
                    }
                    IconButton(onClick = { /* archive */ }) {
                        Icon(Icons.Default.Archive, contentDescription = "Archive")
                    }
                }
            )
        }
    ) { padding ->
        // Your content
    }
    ```
  </TabItem>
  <TabItem label="Custom (raw)">
    ```kotlin
    ContextualTopAppBar(
        selectedCount = selectedIds.size,
        onClearSelection = { selectedIds = emptySet() },
        defaultContent = {
            TopAppBar(title = { Text("Photos") })
        },
        contextualContent = { count ->
            Row(verticalAlignment = Alignment.CenterVertically) {
                Text("$count selected", modifier = Modifier.weight(1f))
                IconButton(onClick = { /* share */ }) {
                    Icon(Icons.Default.Share, contentDescription = "Share")
                }
            }
        }
    )
    ```
  </TabItem>
</Tabs>

## How It Works

The transition is driven by `selectedCount`. When `selectedCount > 0`, the contextual bar fades in. When it returns to `0`, the default bar fades back in.

| State | `selectedCount` | Displayed bar |
|-------|----------------|---------------|
| Nothing selected | `0` | `defaultBar` / `defaultContent` |
| Items selected | `> 0` | Contextual bar with count + actions |
